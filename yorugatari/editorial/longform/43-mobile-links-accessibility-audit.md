## モバイル表示

### 画面幅への対応

共通CSSには次の段階がある。

- 900px超：作品本文とサイド情報を2列表示
- 900px以下：作品本文とサイド情報を1列へ変更し、サイド情報の固定表示を解除
- 一覧カード：3列から2列へ変更
- 600px以下：一覧カードを1列へ変更
- 本文余白を`24px 20px`へ縮小
- フッターリンクを2列グリッドへ変更
- 運営ページの定義リストを1列へ変更

本文コンテナには固定幅を使用せず、`minmax`、`clamp`、`width:min()`を使用している。長いURLや英数字は`overflow-wrap:anywhere`で折り返す。

### 横方向のはみ出し対策

- 本体の最小対応幅は320px
- ページ左右余白は600px以下で12pxずつ
- ヘッダーナビゲーションは900px以下で横スクロール可能
- 横スクロール対象のナビゲーションは本文レイアウトを押し広げない
- ボタン群は折り返し可能
- ストーリー本文は1列へ落ちるため、サイドバーによる最小幅超過はない

`overflow-x:hidden`で問題を隠す処理は使用していない。要素自体が縮小・折り返し・1列化する構造を維持した。

### 文字サイズと読書幅

- 本文はデスクトップ`1.08rem`、600px以下`1.05rem`
- モバイル本文の行高は`1.9`
- 本文段落の最大行長は`42em`
- 見出しは`clamp()`で画面幅に応じて縮小
- `maximum-scale`や`user-scalable=no`を指定せず、ブラウザの拡大を妨げない

## キーボード操作

### スキップリンク

全12ページの`body`直後へ次を静的に追加した。

```html
<a class="skip-link" href="#story-content">本文へ移動</a>
```

通常時は画面外に置き、Tabフォーカス時には画面上部へ表示する。リンク先は次の主コンテンツである。

```html
<main id="story-content" tabindex="-1">
```

JavaScript側にも同じ要素を補うフォールバックを置き、将来のページ追加で静的マークアップが不足した場合にも重複せず補完する。

### フォーカス表示

全リンク、ボタン、入力欄に対し、`:focus-visible`で3pxの明示的な輪郭を表示する。マウス操作時に不要な輪郭を常時表示せず、キーボード操作時の位置を失わない。

### 操作領域

- 一般ボタン：最小高さ44px
- ヘッダーナビゲーション：最小高さ44px
- 検索欄：最小高さ48px
- カテゴリチップ：最小高さ44px
- 関連作品リンク：最小高さ54px

小画面では、サイドバー直下の戻るボタンを横幅100％にする。

## 操作要素

### 解説

解説ボタンへ次を静的に設定した。

- `type="button"`
- `aria-controls="explanation"`
- `aria-expanded="false"`

JavaScript有効時は解説領域へ`hidden`を設定し、開閉時に`hidden`、表示クラス、`aria-expanded`、ボタン文言を同時更新する。CSSだけで非表示にする状態から、アクセシビリティツリー上の状態も一致する構造へ変更した。

JavaScript無効時は解説ボタン自体を非表示とし、解説本文を通常表示する。情報へアクセスできなくなる構造にはしていない。

### お気に入り

お気に入りボタンへ次を設定した。

- `type="button"`
- 初期値`aria-pressed="false"`
- 保存状態に応じて`aria-pressed`を更新
- 表示文言を「お気に入り」「お気に入り済み」で更新

保存領域が利用できない場合も例外で操作全体が停止しない。

### 共有

共有ボタンへ`type="button"`を設定した。端末共有、クリップボード、手動コピーの順で代替手段を持ち、共有APIがなくてもリンクを取得できる。

### 読書進捗バー

読書進捗バーは視覚的補助であり、内容を持たないため`aria-hidden="true"`を設定した。スクロール処理は`passive`リスナーを使用する。

## 見出しと読み上げ

### 見出し階層

各作品ページには作品名の`h1`が1件ある。

サイド情報の「作品情報」と「次に読む」は以前`h3`だったため、`h1`から`h3`へ飛んでいた。両方を`h2`へ変更し、共通CSSを`h2`にも適用した。

### 装飾文字

関連作品リンク先頭の`›`は意味を持たないため、`aria-hidden="true"`を設定した。

怖さの黒丸・白丸表示には、JavaScriptに依存せず次の形式でラベルを付けた。

```html
<span class="fear" aria-label="怖さ 4/5">●●●●○</span>
```

### ナビゲーション

- ヘッダー：`aria-label="主要メニュー"`
- フッター：`aria-label="運営情報"`

運営・編集方針、プライバシー、利用規約、問い合わせへのリンクは、以前はJavaScriptで追加していた。全12ページへ静的に配置し、JavaScript無効時も利用できるようにした。JavaScriptは既存のフッターリンクを検出し、重複追加しない。

## 動きと表示効果

`prefers-reduced-motion: reduce`では次を停止または最小化する。

- スムーズスクロール
- ボタン・カードの移動
- トランジション
- スキップリンクの移動アニメーション

固定ヘッダーには`safe-area-inset-top`、フッターには`safe-area-inset-bottom`を適用し、ノッチ付き端末の安全領域を考慮している。
